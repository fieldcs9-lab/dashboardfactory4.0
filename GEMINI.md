# ข้อบังคับและรายละเอียดโปรเจค (mcDashboard)

## กฎเหล็กประจำโปรเจค
1. **ก่อนเริ่มการปรับปรุงแก้ไขโค้ด หรือดำเนินการใดๆ ที่เกี่ยวข้องกับโปรเจคนี้ ผม (Dwayne Johnson) จะต้องสรุปคำสั่งที่ได้รับมอบหมายตามความเข้าใจออกมาเป็นข้อๆ เพื่อให้ผู้ใช้งานตรวจสอบความถูกต้องก่อนเริ่มงานทุกครั้ง**
2. **ต้องสร้างโฟลเดอร์ `backups` และทำการสำรองไฟล์เดิม (Backup) ไว้ในโฟลเดอร์ดังกล่าวก่อนการแก้ไขโค้ดทุกครั้ง เพื่อความปลอดภัยของข้อมูล**
3. **หากต้องการทราบข้อมูลเชิงลึก รายละเอียด หรือโครงสร้างเกี่ยวกับโรงงานและเครื่องจักร ให้ไปค้นหาและอ้างอิงข้อมูลจากไฟล์ `FACTORY.md` เสมอ**
4. **เมื่อมีการสร้างไฟล์เอกสารนามสกุล Markdown (`.md`) จะต้องตั้งชื่อไฟล์ด้วยตัวพิมพ์ใหญ่ทั้งหมด (UPPERCASE) ทุกครั้ง**

## บริบทของโปรเจค (Project Context)
- **ชื่อโปรเจค:** mcDashboard
- **เทคโนโลยีที่ใช้:** HTML, CSS (Vanilla) และ JavaScript (เน้นความเรียบง่าย ไม่ซับซ้อน)
- **เป้าหมาย:** แดชบอร์ดที่ทันสมัยและสะอาดตา สำหรับแสดงสถานะเครื่องจักร 5 เครื่อง (ข้อมูล: Current, Vibration, Amp)
- **สไตล์การออกแบบ:** เน้นความมินิมอล (Minimalist) สวยงาม และสบายตา (Comfortable for the eyes)

## ประวัติการดำเนินงาน
1. **สร้าง Dashboard:** ออกแบบและเขียนโค้ด `index.html` สำหรับติดตามสถานะเครื่องจักร 5 เครื่อง พร้อมระบบจำลองข้อมูลแบบ Real-time
2. **กำหนดขั้นตอนการทำงาน:** บันทึกข้อบังคับ "สรุปคำสั่งก่อนเริ่มงาน" (Summary-First workflow) ลงในไฟล์การตั้งค่าเฉพาะของโปรเจค
